# 重构计划：移除 A2A 基础设施，添加简化的消息注入端点

## 背景

当前 A2A 系统（Agent-to-Agent）设计用于多 Agent 协作场景，包含：
- Agent 服务发现（AgentCardRegistry）
- 复杂的消息协议（ACK/RESULT 两阶段确认）
- Conversation 跟踪和超时重试
- Redis 分布式消息总线

但实际需求仅是：**向正在运行的 Agent 注入一条消息**。

## 目标

用极简的 `POST /api/agents/{instanceId}/inject` 端点替代整个 A2A 系统。

## 第一阶段：新增 inject 端点

### 1.1 在 Agent 类添加 `injectMessage()` 方法

**文件**: `libs/agent-lib/src/core/agent/agent.ts`

新增公开方法：
```typescript
public async injectMessage(text: string): Promise<void> {
  const userMessage: Message = {
    role: 'user',
    content: [{ type: 'text' as const, text }],
    ts: Date.now(),
  };
  await this.addMessageToMemory(userMessage);

  if (this._status === AgentStatus.Sleeping) {
    this._status = AgentStatus.Running;
    void this.requestLoop();
  }
}
```

### 1.2 在 AgentRuntime 添加 inject 方法

**文件**: `libs/agent-lib/src/core/runtime/AgentRuntime.ts`

新增方法：
```typescript
async injectMessage(instanceId: string, message: string): Promise<void> {
  const agent = await this.getAgent(instanceId);
  if (!agent) throw new Error(`Agent not found: ${instanceId}`);
  await agent.injectMessage(message);
}
```

### 1.3 在 agent.ts 路由添加端点

**文件**: `apps/swarm-runtime/src/routes/agent.ts`

新增端点：
```typescript
fastify.post(
  '/:instanceId/inject',
  {
    schema: {
      tags: ['agents'],
      description: 'Inject a message into an agent',
      body: {
        type: 'object',
        required: ['message'],
        properties: {
          message: { type: 'string', description: 'Message to inject' },
        },
      },
      response: { 202: responseSchema, 404: responseSchema },
    } as any,
  },
  async (request: any, reply: any) => {
    const { instanceId } = request.params;
    const { message } = request.body;
    try {
      const resolvedId = fastify.agentRuntime.resolveAgentId(instanceId);
      await fastify.agentRuntime.injectMessage(resolvedId, message);
      return reply.code(202).send({
        success: true,
        data: { instanceId: resolvedId, status: 'message_injected' },
      });
    } catch (error) {
      return reply.code(404).send({
        success: false,
        error: error instanceof Error ? error.message : String(error),
      });
    }
  },
);
```

## 风险和注意事项

1. **Prisma 迁移**: 如果数据库已有 A2AConversationLog 数据，需要先备份或迁移
2. **并发 Agent 消息**: `injectMessage` 是简单设计，不处理并发注入的排队问题
3. **bib-max-api 改造**: 需要确保新的 HTTP 调用超时处理合理
4. **其他未知依赖**: 执行重构前需 grep 确认无遗漏的 A2A 引用
