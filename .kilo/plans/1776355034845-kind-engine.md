# Plan: Migrate Jupyter Backend as Python Sub-project

## Goal

Create a lightweight Jupyter backend server at `python-apps/jupyter-server/` that provides:
1. Standard REST API for notebook/file management (Contents API)
2. Kernel lifecycle management (start/stop/restart/interrupt)
3. **Synchronous REST API for code execution** (`POST /api/execute`) — wraps `jupyter_client.execute_interactive()` internally
4. No frontend UI (pure API server)

## Architecture Decision

**Do NOT use Jupyter Server (jupyter_server) as dependency.** Instead, build a lightweight FastAPI server that uses `jupyter_client` directly for kernel management and code execution. Rationale:

- Jupyter Server is a heavy dependency (tornado, jupyterlab_server, notebook_shim, etc.) designed for serving web UIs
- For agent integration, we only need: kernel lifecycle + code execution + file management
- `jupyter_client` (already installed: 8.8.0) provides all kernel communication via `KernelManager` + `BlockingKernelClient.execute_interactive()`
- FastAPI is async, lightweight, and aligns with the monorepo's existing Python ecosystem

## API Design

### 1. `POST /api/execute` — Synchronous Code Execution

The core endpoint for agent integration.

**Request:**
```json
{
  "code": "import numpy as np\nnp.array([1,2,3]).sum()",
  "kernel_id": null,          // optional: reuse existing kernel
  "timeout": 30,              // optional: execution timeout in seconds
  "kernel_name": "python3"    // optional: kernel spec name (for new kernel)
}
```

**Response (success):**
```json
{
  "status": "ok",
  "kernel_id": "uuid",
  "execution_count": 1,
  "stdout": "",
  "stderr": "",
  "result": "6",
  "outputs": [
    {"type": "stream", "name": "stdout", "text": "..."},
    {"type": "execute_result", "data": {"text/plain": "6"}, "execution_count": 1}
  ]
}
```

**Response (error):**
```json
{
  "status": "error",
  "kernel_id": "uuid",
  "execution_count": 1,
  "stdout": "",
  "stderr": "",
  "result": null,
  "error": {
    "ename": "ZeroDivisionError",
    "evalue": "division by zero",
    "traceback": ["..."]
  }
}
```

**Implementation:** Uses `jupyter_client.KernelManager` + `BlockingKernelClient.execute_interactive()` with a custom `output_hook` to capture all iopub messages. Runs kernel I/O in a thread pool executor to avoid blocking the async event loop.

### 2. `GET /api/kernels` — List Kernels

**Response:**
```json
{
  "kernels": [
    {"id": "uuid", "name": "python3", "status": "alive", "created_at": "..."}
  ]
}
```

### 3. `POST /api/kernels` — Start Kernel

**Request:**
```json
{
  "name": "python3"
}
```

**Response:**
```json
{
  "id": "uuid",
  "name": "python3",
  "status": "alive"
}
```

### 4. `DELETE /api/kernels/{kernel_id}` — Shutdown Kernel

**Response:** `204 No Content`

### 5. `POST /api/kernels/{kernel_id}/restart` — Restart Kernel

**Response:**
```json
{
  "id": "uuid",
  "name": "python3",
  "status": "alive"
}
```

### 6. `POST /api/kernels/{kernel_id}/interrupt` — Interrupt Kernel

**Response:** `204 No Content`

### 7. `GET /api/health` — Health Check

**Response:**
```json
{
  "status": "ok",
  "kernel_count": 2,
  "version": "0.1.0"
}
```

## Key Implementation Details

### Kernel Pool Manager (`kernel/manager.py`)

- Singleton `KernelPool` that manages multiple `KernelManager` instances
- Each kernel runs in its own subprocess (via `jupyter_client.KernelManager`)
- Thread-safe operations using `asyncio.Lock`
- Auto-cleanup of dead kernels
- Configurable max kernel limit and idle timeout

### Code Executor (`kernel/executor.py`)

- `execute_code(kernel_id, code, timeout)` function
- Uses `BlockingKernelClient.execute_interactive()` with custom `output_hook`
- Runs in `asyncio.get_event_loop().run_in_executor()` to avoid blocking
- Captures: stdout, stderr, execute_result, display_data, error
- Returns structured `ExecutionResult` Pydantic model
- If `kernel_id` is None, creates a new kernel automatically

### Async Bridge

Since `jupyter_client` is synchronous (ZMQ-based), all kernel operations must run in a thread executor:

```python
async def execute_code_async(kernel_id: str, code: str, timeout: float = 30):
    loop = asyncio.get_event_loop()
    return await loop.run_in_executor(None, _execute_sync, kernel_id, code, timeout)
```

### FastAPI App (`app.py`)

- `lifespan` context manager: initialize kernel pool on startup, shutdown all kernels on exit
- CORS middleware enabled (for agent integration)
- Structured error handling with proper HTTP status codes

## Workspace Integration

### uv workspace

Add to root `pyproject.toml`:
```toml
[tool.uv.workspace]
members = ["python-libs/*", "python-apps/*", "apps/swarm-fs", "apps/file-renderer"]
```

```bash
cd python-apps/jupyter-server
uv run uvicorn jupyter_backend.app:create_app --factory --host 0.0.0.0 --port 8888
```
